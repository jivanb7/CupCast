# CupCast

**Football match prediction service, served as a web app.**

CupCast forecasts the outcome of upcoming matches across ten football leagues, surfaces value picks where our model has high conviction, and tracks our live accuracy on a public model performance page so anyone can audit how the model is actually doing.

🌐 **Live demo:** [https://cupcast.app](https://cupcast-frontend-dlotj2g4dq-uw.a.run.app/login)

## What CupCast does

There are a lot of football prediction sites. Most of them are black boxes. They show you a probability and ask you to trust it.

CupCast is built around the opposite idea: **show your work**.

Every prediction is generated by an XGBoost model trained on 65,000 historical matches and 87 engineered features. Every retrain is benchmarked against the live model and only ships if it wins. Every match the model predicts is scored against the actual result the moment the game ends, on a page anyone can read. If the model is wrong, that shows up in public, in real time.

You can use CupCast to:

- **See match predictions** for upcoming games across ten leagues — top-five European leagues, the English lower divisions, MLS, the Champions League, and the World Cup.
- **Find value picks** that flag matches where our model has the strongest disagreement with consensus.
- **Audit live model performance** — rolling accuracy, log loss, per-league breakdown, and recent versus prior comparison, all on a public page.
- **Browse match details** with feature-level reasoning behind each prediction.

## How it works (the short version)

```
Multiple data sources  →  Postgres (Supabase)  →  XGBoost via MLflow  →  FastAPI  →  React app
                                                       ↑
                                              GitHub Actions (weekly retrain + CI/CD)
                                                       ↑
                                              Cloud Scheduler (eleven cron jobs)
```

- **Data layer**: A historical match corpus going back two decades, plus live odds, lineups, injuries, top scorer data, and real-time score polling on matchday. All of it lands in Postgres on Supabase.
- **Model**: XGBoost with isotonic calibration, trained weekly on 87 features. Recency-weighted with a three-year half-life so recent seasons dominate. Tracked end-to-end in MLflow on a Compute Engine VM.
- **Promotion gate**: A new model only ships if it beats the live champion on both validation log loss and validation accuracy. Otherwise, the current model stays in production.
- **Backend**: FastAPI on Cloud Run, eight routers, sub-second inference.
- **Frontend**: React + Vite on Cloud Run.
- **CI/CD**: GitHub Actions. We authenticate to Google Cloud using Workload Identity Federation, so there's no long-lived service account JSON in the repo. Every push to main triggers tests, runs database migrations, builds Docker images, and deploys with a smoke test and automatic rollback.
- **Orchestration**: Eleven Cloud Scheduler jobs, with cadences ranging from every minute (live score sync during matches) to weekly (model retrain).

## Project status

CupCast is fully deployed and running in production. Predictions update daily, model retrains run every Monday with a promotion gate, and the live performance page tracks accuracy continuously. The current production model is `v0.4.0`, an XGBoost classifier with isotonic calibration trained on April 27, 2026.
