## What broke and how it was caught

> [!warning] Production outage — Apr 24, ~22:50 PT
> First push to main went live, **production backend crashed** with `column matches.stage does not exist`. Root cause: deploy workflow had no migration step. Schema lived in code but was never applied to prod Postgres.

**Recovery (8 minutes)**:
1. Dropped 3 empty auto-created tables (created by `Base.metadata.create_all` on container startup)
2. Stamped Alembic at initial schema `4922171d5711`
3. Ran `alembic upgrade head` → 7 migrations applied cleanly
4. Backend recovered to 200 OK
5. Ran 6 backfill scripts on prod DB
6. Generated predictions + odds + Monte Carlo

**Hardening so it can't happen again** — see [[PRODUCTION-RUNBOOK]] for the full multi-layer defense.
